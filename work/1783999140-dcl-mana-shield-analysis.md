## Native contract

The reaction unit is `qword[0x14186AF68]`. Its target-local result record is
`qword[0x14186AF70] = unit+0x1BE`, so record `+0x06/+0x0A/+0x27/+0x10` are unit
`+0x1C4/+0x1C8/+0x1E5/+0x1CE` respectively.

When reaction id 445 is equipped, staged HP debit is nonzero, and current MP is merely
nonzero, the native branch:

1. adds the complete staged HP debit to staged MP debit;
2. caps the staged MP debit at 999;
3. clears result bits `0x80` and `0x20`, then sets MP-result bit `0x20`;
4. clears the staged HP debit; and
5. stages exact reaction id `445` at unit `+0x1CE`.

The later apply path independently clamps `currentMP + credit - debit` into
`0..MaxMP`; it never restores HP damage when the MP debit exceeds current MP. Therefore
**one remaining MP point can natively prevent the entire HP hit**. This is a Strong
current-build engine fact, not a balanced proportional shield.

## DCL implementation boundary

The existing combined pre-clamp formula context already exposes `dcl.oldMpDebit`,
`target.mp`, `target.maxMp`, staged HP/MP formulas, and the current action context. It can
calculate a bounded authored HP/MP split. A complete replacement also needs coherent
result-flag/presentation ownership when both HP and MP change or when a native redirect is
rejected for insufficient MP. Ratio and all-or-partial absorption remain design/calibration
decisions; the native one-MP full shield must not be mistaken for the final DCL rule.

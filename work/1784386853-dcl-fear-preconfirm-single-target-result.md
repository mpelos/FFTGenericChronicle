[DCL-FEAR-TARGET] casterIdx=17 type=0x01 ability=0 state=0x2F owned=0 affected=1 opposing=0 decision=allow

Basic Attack legitimately uses actor action/presentation id `0`, so those fields do not falsify the
actor identity. The decisive result is `actorTargetCount=0` before confirmation followed by one
expanded target in execution. `actor+0x1A9/+0x1AA` is therefore not materialized early enough to
authorize player confirmation. The forecast cache also has no exact builder decision at this point,
which explains its default `casterIdx=0/type=0` row and fail-open result.

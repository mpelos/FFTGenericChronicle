## Resolver call contract

The weapon-target evaluator obtains the acting unit index from the 20-byte order context,
selects the equipped hand, reads the weapon AttackFlags, and resolves the candidate unit at
the staged target coordinates. The projectile calls use:

```text
rcx = acting unit index
rdx = pointer to staged target coordinates (unit/order +0x1AC, +0x1AE, +0x1B0)
r8d = candidate unit index resolved at those coordinates
eax = projectile resolver's actual reached/intercepted unit index, or a negative failure
```

For both Arc and Direct, the call site accepts the candidate only when both indices are
nonnegative and `candidate == resolver_result`. This equality is the native authoritative
target-inclusion/line-of-fire verdict for the selected weapon trajectory.

## Static interpretation

- **Strong:** the engine has a native weapon-specific line-of-fire authority, selected from
  the weapon's `AttackFlags`; Euclidean/Manhattan distance is not a substitute.
- **Strong:** Arc and Direct return the actually reached/intercepted unit index. Comparing
  that result with the intended source is the authoritative inclusion test needed by
  Countershot and other returned weapon attacks.
- **Strong:** the Arc resolver is real code and performs trajectory/collision work; the Direct
  resolver immediately enters protected VM code, so its internal obstacle algorithm cannot
  be reconstructed completely offline from aligned real code.
- **Strong:** `0x2804D8` is not generic projectile LoS. It is selected specifically by the
  `Lunging` flag and rejects non-cardinal paths while recursively testing intermediate terrain.
- **Implementation boundary:** invoking Arc/Direct from an unrelated poller is unsafe because
  both participate in mutable targeting globals. Reuse must occur synchronously on the game
  thread at a native targeting/Reaction boundary, or the DCL must let a synthesized native
  basic weapon order run the same resolver. A dedicated observe-only proof must establish
  which option preserves target/source identity without redirecting into an interceptor.

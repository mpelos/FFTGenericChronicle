# Live Gate Plan

Generated by `tools/report_live_gate_plan.py`. Do not hand-edit.

This is the ordered live checklist for the flexible battle-mechanics code mod. The
offline gate proves the repo is prepared; these steps collect the evidence that cannot
be produced without running FFT through Reloaded-II.

## Safety Rules

- Start every session with `powershell -ExecutionPolicy Bypass -File codemod\run-offline-checks.ps1`.
- Close Reloaded-II and `FFT_enhanced` before running any non-`-DryRun` preparation helper.
- Keep `MinHpFloor=1` for custom lethal formulas. Do not use legacy HP=0/KO-bit profiles as
  the active death path.
- Use high-HP targets for sentinel calibration and neuter-gap checks because vanilla placeholder
  damage lands before the runtime rewrite.
- Archive or replace stale `battleprobe_log.txt` before each gate; the helpers do this when
  run without `-NoArchiveLog`.

## Gate 1 - Custom Formula Demo

Purpose: prove a real formula reads the resolved attacker and target in live battle.

Prepare:

```powershell
powershell -ExecutionPolicy Bypass -File codemod\prepare-custom-formula-demo.ps1 -DryRun
powershell -ExecutionPolicy Bypass -File codemod\prepare-custom-formula-demo.ps1
```

In game:

- Hit the same target with two attackers that have different PA.
- Hit two targets with different Faith using the same attacker.
- Optional: trigger one counterattack to exercise `counter-inversion`.

Watch:

```powershell
python tools\watch_live_mapping.py --runtime-events 3 --ct-runtime-attackers 1 --require-trace-var trace.attackerpa --require-trace-var trace.targetfaith --require-trace-var trace.attackersourcect --require-trace-var trace.finaldamage --placeholder-rewrites 3 --max-placeholder-damage 30 --max-rewrite-failures 0
python tools\analyze_battleprobe_log.py
```

Counter add-on:

```powershell
python tools\watch_live_mapping.py --runtime-events 1 --counter-runtime-attackers 1 --require-trace-var trace.attackersourcecounter --max-rewrite-failures 0
```

Pass evidence: `[RUNTIME]` attacker source is `ct-reset` or `counter-inversion`, trace vars
include attacker PA and target Faith, and `[REWRITE]` final damage changes with those values.

## Gate 6 - Same-Hit KO Research

Purpose: only revisit same-hit custom KO after Gate 5 exposes a plausible pre-damage or
current-action context. Direct HP=0 and `+0x61|=0x20` writes are refuted.

No safe offline or blind-live command exists for this gate. The current accepted behavior is
`MinHpFloor=1`: custom lethal results leave the target at 1 HP, then the engine owns real KO
on a later vanilla chip.
